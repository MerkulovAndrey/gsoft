Практическое задание

Небольшое web-приложение – личный кабинет для каждого из типов пользователей (менеджеры и клиенты) по следующему техническому заданию:

  1. Основное окно – таблица грузов со всеми необходимыми связями - id_client должен быть заменен на name из соответствующие таблицы клиентов. Вместо id_manager должно выводится полное имя (имя и фамилия) менеджера. Каждый клиент должен видеть только свои грузы, каждый менеджер должен видеть грузы тех клиентов, которых он ведет.
  2. При нажатии на ячейку клиента или менеджера открывается окно детализации (полная информация по менеджеру или клиенту).
  3. Таблицу, какой ее видит клиент или менеджер, можно выгрузить в Excel прямо в браузере или отравить Excel-файл по почте (т.е. сначала формируется файл, потом отправляется на почту).
  4. Клиент в отдельном окне «Завести груз» может добавить в систему новый груз. Указывается только номер контейнера. Status по дефолту Awaiting.
  5. Менеджер в отдельном окне «Новые грузы» может назначить себя исполнителем по любому из только что созданных грузов.
  6. По доступной только менеджеру ссылке «Редактировать груз» можно менять статус груза (On board или Finished) и время прибытия.
  7. Для выгрузки в Excel рекомендуется использовать библиотеку PHP-Excel. Использовать мейнстрим-фреймворки или CMS нельзя (но можно MVC, как основу архитектуры проекта). Оценивается Ваше собственное видение решения задачи и навыки проектирования. Верстка значения не имеет. Пользоваться JS/JQuery можно. Рекомендуется после каждого итерационного цикла выкладывать проект в git-репозиторий.
